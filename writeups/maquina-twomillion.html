Resumen Ejecutivo

Esta máquina presenta una vulnerabilidad en una API web que permite escalada de privilegios a través de la manipulación de parámetros JSON. Posteriormente, se explota CVE-2023-4911 (Looney Tunables) para obtener acceso root mediante un buffer overflow en glibc.

Cadena de Ataque:

  1. Reconocimiento → Descubrimiento de API
  2. Ingeniería inversa de código JavaScript ofuscado
  3. Generación de código de invitación mediante API
  4. Escalada de privilegios a admin via manipulación de API
  5. Command Injection para obtener reverse shell
  6. Explotación de CVE-2023-4911 para escalada a root

Introduccion

En este documento se presenta la explotación completa de la máquina TwoMillion, una máquina basada en Linux (Ubuntu), la cual simula un entorno web corporativo con múltiples fallos de seguridad encadenables. A lo largo del análisis se demuestra cómo una serie de malas prácticas en el diseño de la aplicación, combinadas con una vulnerabilidad crítica a nivel de sistema, permiten comprometer completamente el servidor hasta obtener acceso root.

La intrusión inicia con la enumeración de una API web vulnerable, cuya lógica de control de acceso es defectuosa, permitiendo la escalada de privilegios mediante manipulación de parámetros JSON. Posteriormente, se aprovecha una inyección de comandos en un endpoint administrativo para obtener acceso inicial al sistema. Finalmente, se explota la vulnerabilidad CVE-2023-4911 (Looney Tunables) en glibc, logrando una escalada total de privilegios.

Este laboratorio pone de manifiesto la importancia de:

  • La correcta validación de datos en APIs.
  • La implementación rigurosa de controles de acceso.
  • La gestión segura de credenciales.
  • El mantenimiento actualizado de librerías críticas del sistema.

Además, permite reforzar habilidades prácticas en pentesting web, explotación de APIs, command injection, post-explotación en Linux y escalada de privilegios a nivel de sistema.

Tipo de Vulnerabilidad:

  • OWASP A01:2021 - Broken Access Control

Causa Raíz:

Impacto:

  • Cualquier usuario autenticado puede convertirse en administrador
  • Acceso completo a funcionalidades administrativas
  • Potencial para comprometer completamente la aplicación

6.3 Prueba de Concepto - Command Injection

6.3.1 Hipótesis de Vulnerabilidad

Los generadores de VPN típicamente ejecutan comandos como:

openvpn --genkey --secret /tmp/username.key

Si el parámetro username no está sanitizado, podríamos inyectar comandos adicionales usando operadores de shell como:

  • && - Ejecuta el siguiente comando si el anterior tuvo éxito
  • ; - Ejecuta comandos secuencialmente
  • | - Pipe para encadenar comandos

6.3.2 Preparación del Listener

En nuestra máquina atacante, configuramos un listener con netcat o penelope:

penelope -p 4444

Características de Penelope:

  • Auto-upgrade de shell a PTY
  • Persistencia de sesión
  • Transferencia de archivos integrada
  • Soporte multi-sesión

6.6 Análisis de la Vulnerabilidad de Command Injection

Tipo de Vulnerabilidad:

  • CWE-78: OS Command Injection
  • OWASP A03:2021 - Injection

Código Vulnerable (Hipotético):

<?php
// VULNERABLE - NO HACER ESTO
$username = $_POST['username'];
$command = "openvpn --genkey --secret /tmp/" . $username . ".key";
exec($command);
?>

Impacto:

  • Ejecución remota de código (RCE)
  • Compromiso completo del servidor web
  • Acceso a datos sensibles
  • Posibilidad de pivoting en la red interna Mitigación:
  1. Validar entrada con whitelist de caracteres permitidos
  2. Usar funciones seguras (evitar exec(), system(), shell_exec())
  3. Implementar escapado de caracteres especiales con escapeshellarg()
  4. Ejecutar servicios web con privilegios mínimos (principio de mínimo privilegio)
  5. Usar contenedores o sandboxing para aislar la aplicación

CVE-2023-4911 — Looney Tunables

Afecta a: glibc (GNU C Library)
Año: 2023
Impacto: Escalada de privilegios a ROOT
Tipo: Buffer Overflow en variables de entorno

¿QUÉ ES EXACTAMENTE?

Es una vulnerabilidad crítica en glibc que permite que un usuario normal obtenga ROOT, explotando cómo se procesan ciertas variables de entorno llamadas GLIBC_TUNABLES.

El fallo ocurre ANTES de que el programa baje privilegios, lo cual es lo más peligroso posible.

COMPONENTE VULNERABLE

El fallo está en cómo glibc procesa esta variable:

GLIBC_TUNABLES

Esta variable se evalúa antes de que el binario quite privilegios, incluso en binarios SUID root como:

  • sudo
  • su
  • passwd
  • pkexec

TIPO DE VULNERABILIDAD

Es un:

Stack Buffer Overflow por mala validación de entrada

Puedes sobrescribir estructuras internas de glibc y redirigir la ejecución.

9. Mitigaciones Recomendadas

9.1 Vulnerabilidades de la Aplicación Web

  1. API de Invitaciones:
    • Implementar rate limiting
    • Validar códigos del lado del servidor
    • No exponer lógica de generación en el cliente
  2. Escalada de Privilegios via API:
    • Validar permisos en cada endpoint
    • No permitir modificación directa del flag is_admin
    • Implementar autenticación basada en roles (RBAC)
  3. Command Injection:
    • Sanitizar todas las entradas de usuario
    • Utilizar funciones seguras (evitar system(), exec())
    • Implementar whitelisting de caracteres permitidos
  4. Exposición de Credenciales:
    • No almacenar credenciales en archivos .env accesibles
    • Utilizar gestores de secretos
    • Aplicar principio de mínimo privilegio

9.2 Sistema Operativo

  1. CVE-2023-4911:
    • Actualizar glibc a la versión más reciente
    • Aplicar parches de seguridad regularmente
    • Monitorear advisories de seguridad

bash

# Verificar versión de glibc
ldd --version

# Actualizar sistema
sudo apt update && sudo apt upgrade

10. Conclusión

La máquina TwoMillion demuestra la importancia de:

  • Validación adecuada de entrada en APIs
  • Implementación correcta de controles de acceso
  • Mantenimiento actualizado de componentes del sistema
  • No exponer información sensible en archivos de configuración
